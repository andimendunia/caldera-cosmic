1. Infrastruktur & Persyaratan Sistem

1.1 Laragon

Caldera berjalan di Laragon, sebuah lingkungan development portabel untuk Windows yang menyediakan stack web development lengkap.

Komponen Laragon (harus dijalankan):

  • Apache Web Server - Hosting aplikasi Laravel
  • MySQL Database Server - Menyimpan semua data aplikasi
  • PHP 8.2+ - Runtime environment untuk Laravel
Panel kontrol Laragon menampilkan layanan Apache dan MySQL yang sedang berjalan
Penting: Apache dan MySQL adalah komponen terintegrasi dalam Laragon dan harus dijalankan melalui panel kontrol Laragon sebelum menjalankan Caldera.

1.2 Caldera Daemon Manager (CDM)

Caldera Daemon Manager adalah aplikasi GUI berbasis Python yang mengelola dan memantau program background (daemon) penting yang diperlukan untuk operasi Caldera.

Fitur CDM:

  • Manajemen dan monitoring layanan otomatis
  • Viewing dan analisis log real-time
  • Monitoring status layanan dengan kemampuan restart otomatis
  • Manajemen konfigurasi melalui antarmuka GUI

Lokasi CDM:

py/daemon-manager/dist/cdm_v1.0/cdm.exe
Interface GUI Caldera Daemon Manager menampilkan status layanan dan kontrol management

Opsi Manajemen CDM:

  • Aplikasi Standalone: Jalankan executable CDM secara langsung
  • Interface Web: Akses melalui interface Caldera di Menu Akun > Administrasi > Kelola Daemon (khusus superuser)
Interface web daemon manager dalam aplikasi Caldera untuk superuser

1.3 Layanan Inti

CDM mengelola layanan background kritis berikut yang harus berjalan terus-menerus:

Queue Worker

php artisan queue:work

Memproses background job dan tugas antrian yang penting untuk pemrosesan data dan notifikasi.

Schedule Worker

php artisan schedule:work

Menjalankan tugas terjadwal termasuk pembersihan data, kalkulasi, dan rutinitas otomatis.

CLM Polling (InsClmPoll)

php artisan app:ins-clm-poll --d

Daemon polling monitoring temperatur yang mengumpulkan data lingkungan dari gedung IP.

CTC Polling (InsCtcPoll)

php artisan app:ins-ctc-poll --d

Daemon polling kontrol ketebalan kalender untuk monitoring proses calender.

STC Routine (InsStcRoutine)

php artisan app:ins-stc-routine --d

Daemon rutinitas kontrol suhu IP STC dengan Ambient Temperature (AT).

1.4 Tugas Terjadwal

Laravel scheduler menjalankan tugas otomatis pada interval yang ditentukan. Ini didefinisikan dalam routes/console.php:

Pembersihan Data (Harian)

php artisan app:data-cleanup

Sistem pembersihan data terpadu dengan retensi yang dapat dikonfigurasi per modul:

  • OMV: 6 bulan (captures & metrics)
  • CTC: 6 bulan (metrics)
  • RDC: 6 bulan (hasil uji)
  • STC: 6 bulan (log perangkat & ringkasan)
  • CLM: 12 bulan (record lingkungan)
  • LDC: 24 bulan (record hide)
  • RUB: 12 bulan (record rubber batch)
  • INV: 60 bulan (record sirkulasi inventaris)

Sinkronisasi Query Inventaris (Setiap 5 Menit)

php artisan app:sync-inv-query

Menyinkronkan parameter query inventaris pengguna (items, sirkulasi, area) dari sesi ke database untuk konsistensi filter dan pencarian.

1.5 Konfigurasi & Monitoring Data Cleanup

Sistem data cleanup dapat dikonfigurasi dan dipantau untuk memastikan operasi yang optimal.

Mengonfigurasi Data Cleanup

Untuk mengubah periode retensi data per modul:

  1. Edit file: app/Console/Commands/InsDataCleanup.php
  2. Ubah array $retentionConfig (nilai dalam bulan, 0 = tidak pernah cleanup)
  3. Deploy perubahan ke server
private array $retentionConfig = [ 'omv' => 6, // OMV captures and metrics 'ctc' => 6, // CTC metrics 'rdc' => 6, // RDC test results 'stc' => 6, // STC device logs and summaries 'clm' => 12, // CLM environmental records 'ldc' => 24, // LDC hide records 'rubber_batches' => 12, // Rubber batch records 'inv' => 60, // Inventory circulation records ];

Testing Data Cleanup

Sebelum menjalankan cleanup di production, gunakan mode testing:

Dry-Run Mode

php artisan app:data-cleanup --dry-run

Preview apa yang akan dihapus tanpa melakukan penghapusan sebenarnya.

Test Mode

php artisan app:data-cleanup --test

Mode testing dengan jumlah record terbatas untuk pengujian aman.

Combined Testing

php artisan app:data-cleanup --dry-run --test

Kombinasi dry-run dan test mode untuk pengujian yang paling aman.

Monitoring Data Cleanup

Untuk memantau operasi pembersihan data:

  • Daemon Manager: Periksa log di py/daemon-manager/logs/schedule-work.log
  • Output real-time: Lihat statistik penghapusan dan error melalui log files
  • Verification: Gunakan dry-run untuk memverifikasi operasi sebelum execution
Catatan Penting: Selalu gunakan dry-run mode terlebih dahulu untuk memverifikasi operasi cleanup sebelum menjalankan penghapusan data yang sebenarnya di lingkungan produksi.

2. Struktur Proyek & Arsitektur

2.1 Framework & Stack

  • Laravel 12 dengan PHP 8.2+ - Framework inti
  • Livewire Volt - Komponen UI reaktif (tanpa controller tradisional)
  • MySQL - Database utama dengan migrasi ekstensif
  • Vite - Asset bundling dengan TailwindCSS
  • Chart.js - Visualisasi data
  • Lucide - Library ikon untuk ikonografi UI yang konsisten
  • Modbus TCP client - Integrasi peralatan industri

2.2 Modul Aplikasi Inti

Sistem Insights (Monitoring Manufaktur)

OMV (Open Mill Validation)

Monitoring proses open-roll mixing dengan manajemen resep, pelacakan ampere real-time, dan kalkulasi energi.

CTC (Calendar Thickness Control)

Monitoring ketebalan untuk proses calender dengan analitik performa mesin.

STC (Stabilization Temperature Control)

Monitoring dan kontrol proses IP Stabilization Chamber.

RDC (Rheometer Data Collection)

Pengumpulan data testing rheometer untuk kontrol kualitas rubber.

LDC (Leather Data Collection)

Pengumpulan data inspeksi kulit hasil mesin nesting di proses OKC.

DWP (Deep Well Press)

Monitoring proses deep well press di area Assembly.

Sistem Manajemen Inventaris

  • Manajemen stok multi-unit
  • Sistem otorisasi berbasis area
  • Alur sirkulasi (penambahan/pengambilan) dengan proses persetujuan
  • Dukungan multi-mata uang dengan kalkulasi otomatis
  • Analitik lanjutan termasuk aging stok

Sistem Notifikasi

Sistem notifikasi built-in untuk pengumuman fitur dan komunikasi pengguna:

  • Notifikasi berbasis database disimpan dalam tabel notifications
  • Pengumuman fitur melalui perintah artisan FeatureAnnounce
  • Opsi targeting pengguna: superuser, pengguna aktif, pengguna terbaru, pengguna inventaris
  • Dukungan konten kaya dengan ikon (menggunakan library ikon Lucide), teks konten, dan URL yang dapat diklik
  • Post pengumuman untuk penjelasan fitur detail dengan gambar

Mengirim Notifikasi

php artisan app:feature-announce

Perintah interaktif yang memungkinkan Anda untuk:

  • Memilih dari pengumuman fitur yang telah ditentukan
  • Memilih grup pengguna target (superuser, pengguna aktif, pengguna terbaru, dll.)
  • Preview konten notifikasi sebelum mengirim
  • Otomatis membuat post pengumuman untuk konten detail

2.3 Arsitektur Database

  • Development berbasis migrasi dengan file migrasi ekstensif
  • Penamaan tabel berprefix: ins_* (insights), inv_* (inventory)
  • Timestamps pada sebagian besar entitas untuk audit tracking
  • Kolom JSON untuk penyimpanan data fleksibel (komposisi, preferensi)
  • Model otorisasi (*Auth) untuk kontrol izin granular

3. Deployment Server & Perubahan Kode

3.1 Alur Deployment Saat Ini

Karena server Caldera tidak terhubung ke internet, perubahan kode harus melalui PC perantara. Alur kerja melibatkan tiga langkah utama:

  • Langkah A: Development Lokal ke GitHub - Develop dan push perubahan dari PC Anda ke repository pusat
  • Langkah B: GitHub ke PC Pak Khafit - Pull perubahan terbaru ke folder bersama (172.70.66.102/mm/TIM DGT/_Repos/caldera-cosmic)
  • Langkah C: PC Pak Khafit ke Server Caldera - Deploy perubahan dan update environment produksi

3.2 Perintah Deployment

Perintah Langkah A (Lokal ke GitHub):

git add . git commit -m "Pesan commit Anda" git push origin main

Perintah Langkah B (GitHub ke PC Pak Khafit):

git pull origin main

Perintah Langkah C (PC Pak Khafit ke Server Caldera):

git pull mm main php artisan config:cache php artisan route:cache npm run build php artisan migrate:status
Penting: Selalu periksa php artisan migrate:status untuk memastikan semua migrasi database telah diterapkan dengan sukses. Jalankan php artisan migrate jika ada migrasi yang tertunda.

3.3 Alur Deployment Masa Depan

Jika server Caldera mendapat konektivitas internet di masa depan, proses deployment dapat disederhanakan menjadi alur kerja langsung dua langkah:

Langkah Masa Depan 1: Lokal ke GitHub

git add . git commit -m "Pesan commit Anda" git push origin main

Langkah Masa Depan 2: GitHub ke Server Caldera (Langsung)

git pull origin main php artisan config:cache php artisan route:cache npm run build php artisan migrate:status
Pertimbangan Masa Depan: Konektivitas internet langsung akan menghilangkan kebutuhan untuk PC perantara (PC Pak Khafit) dan memungkinkan proses deployment yang lebih efisien, berpotensi termasuk pipeline CI/CD otomatis.

4. Python Utilities

Sistem Caldera menggunakan beberapa utility Python yang mendukung operasi inti. Setiap utility dirancang untuk menangani aspek spesifik dari pipeline data dan manajemen sistem.

4.1 Daemon Manager

Aplikasi GUI berbasis Python untuk mengelola dan memantau layanan background Caldera.

Lokasi & Struktur:

py/daemon-manager/ ├── main.py # File utama aplikasi GUI ├── main.spec # Spec file PyInstaller untuk build executable ├── main_backup.spec # Backup spec file (gunakan jika main.spec terewrite) ├── gui/ # Komponen GUI ├── dist/cdm_v1.0/ # Executable hasil build └── requirements.txt # Dependencies Python

Development & Modification:

  • File untuk diedit: main.py - Entry point dan logika utama aplikasi
  • Komponen GUI: Direktori gui/ berisi modul-modul interface
  • Dependencies: Install dengan pip install -r requirements.txt

Building Executable:

cd py/daemon-manager pyinstaller main.spec
Penting: Jika main.spec tidak terdefinisi atau terewrite dengan nilai default, gunakan main_backup.spec sebagai template. Copy main_backup.spec ke main.spec dan sesuaikan jika diperlukan.

Jumlah Instance

1 Instance - Satu copy tunggal yang mengelola seluruh sistem

Catatan Deployment: CDM harus berjalan di server utama Caldera karena memerlukan akses langsung ke database dan file konfigurasi sistem. Tidak diperlukan instalasi di mesin produksi individual.
Interface utama Caldera Daemon Manager dengan monitoring real-time layanan

Fitur Utama:

4.2 LDC Worker

Server WebSocket FastAPI untuk Leather Data Collection dengan packet sniffing dan broadcasting data real-time.

LDC Worker console menampilkan packet capture dan WebSocket broadcasting

Lokasi & Struktur:

py/ldc-worker/ ├── app.py # Aplikasi FastAPI utama ├── app.spec # PyInstaller spec file ├── requirements.txt # Dependencies Python └── dist/ # Executable hasil build

Fitur Teknis:

  • Framework: FastAPI dengan WebSocket support
  • Packet Sniffing: Menggunakan Scapy untuk monitoring traffic loopback
  • Data Processing: Parsing HTTP POST requests dengan chunked encoding
  • Broadcasting: Real-time data ke connected WebSocket clients
  • Deduplication: Cache untuk mencegah pengiriman data duplikat

Requirements:

  • Administrator privileges: Diperlukan untuk packet capture
  • Dependencies: fastapi, uvicorn, scapy, websockets
  • Port: 32998 (WebSocket endpoint: /ws)

Building Executable:

Untuk membuat executable standalone:

cd py/ldc-worker pyinstaller app.spec

Executable akan tersedia di direktori dist/ setelah build selesai.

Usage:

Silakan merujuk ke instalasi yang sudah ada di mesin dan cara menjalankannya dengan benar.

Deployment & Konfigurasi Instance:

Lokasi Deployment

Comelz Nesting Machines

  • Model: NT/NEK/NEK+
  • Machine XA, XB, XC, XD
  • Total: 4 instance terpisah

Konfigurasi Display

Dual Monitor Setup:

  • Main Display: Comelz Quality Editor app
  • Secondary Display: Waterfox browser (Caldera interface) + LDC Worker (minimized)
Catatan Deployment: LDC Worker harus dijalankan dengan administrator privileges pada setiap mesin Comelz untuk packet capture. Pastikan aplikasi berjalan minimized agar tidak mengganggu operasi Comelz Quality Editor di main display.

Penggunaan Dalam Sistem:

  • Menangkap data POST dari aplikasi leather inspection
  • Konversi area measurements dari mm² ke ft²
  • Broadcasting data ke Caldera untuk real-time monitoring
  • Monitoring kualitas kulit pada 4 mesin secara bersamaan

4.3 OMV Worker

Server Flask API untuk Open Mill Validation dengan komunikasi serial, capture kamera, dan database queuing.

OMV Worker Flask API interface dengan endpoint monitoring

Lokasi & Struktur:

py/omv-worker/ ├── app.py # Aplikasi Flask utama ├── config.json # File konfigurasi ├── config.json.example # Template konfigurasi ├── database.db # Database SQLite untuk queue ├── launcher.bat # Script launcher Windows └── requirements.txt # Dependencies Python

Fitur Teknis:

  • Framework: Flask dengan CORS support
  • Serial Communication: Komunikasi dengan Arduino via serial port
  • Camera Capture: OpenCV untuk capture dan processing gambar
  • Database Queue: SQLite untuk antrian data dengan retry mechanism
  • Time Synchronization: Sinkronisasi waktu dengan server utama

API Endpoints:

GET /get-line

Mengambil nomor line dari konfigurasi

POST /send-data

Mengirim data OMV ke server dengan queuing jika gagal

GET /get-data

Membaca data sensor dari Arduino via serial

GET /get-photo

Capture dan crop gambar dari kamera

Konfigurasi:

File config.json berisi pengaturan untuk:

  • Server endpoints dan IP addresses
  • Serial port settings (port, baud rate, threshold)
  • Camera settings (index, resolution, frame count)
  • Data handling (retry attempts, backoff timing)

Usage:

Silakan merujuk ke instalasi yang sudah ada dan cara menjalankannya dengan benar.

Deployment & Konfigurasi Instance:

Lokasi Deployment

Open-roll Mixing Machines

  • Dipasang pada setiap mesin mixing karet
  • Instance terpisah per mesin
  • Terhubung ke Arduino dan kamera

Konfigurasi Display

Setup Display:

  • Waterfox Browser: Caldera interface
  • OMV Worker: Berjalan minimized di background
  • Akses mudah untuk monitoring proses mixing

4.4 Translation Extract

Tool interaktif untuk manajemen string terjemahan Laravel dengan ekstraksi, pembersihan, dan sinkronisasi fitur.

Translation Extract tool menampilkan proses ekstraksi dan sinkronisasi string terjemahan

Lokasi & Struktur:

py/translation-extract/ ├── app.py # Aplikasi utama └── exception.json # Daftar string yang dikecualikan dari ekstraksi

Fitur Utama:

Extract New Translations

Scan file .blade.php dan .php untuk mencari string __() dan menambahkannya ke file bahasa

Remove Unused Translations

Identifikasi dan hapus key terjemahan yang tidak digunakan lagi dalam kode

Sort Translations

Urutkan semua key terjemahan secara alfabetis dalam file bahasa

Sync Translations

Sinkronisasi key antar semua file bahasa (en.json, id.json, dll.)

Separate Empty Translations

Pisahkan terjemahan kosong ke bagian bawah file untuk kemudahan editing

Penggunaan:

# Jalankan dari root directory proyek Caldera python py/translation-extract/app.py

File Konfigurasi:

  • exception.json: Berisi array string yang dikecualikan dari ekstraksi
  • Target direktori: resources/ dan app/ untuk scanning
  • Output file: lang/en.json sebagai file utama
Catatan: Tool ini harus dijalankan dari root directory proyek Caldera untuk memastikan path relatif yang benar ke direktori resources/, app/, dan lang/.

Deployment & Konfigurasi Instance:

Lokasi Deployment

Development Machines

  • PC developer atau PC Pak Khafit
  • Tidak perlu instalasi permanen
  • Akses ke codebase diperlukan

Frequency & Usage

On-Demand Basis

  • Hanya dijalankan saat update bahasa
  • Tidak perlu berjalan background
  • Tool maintenance berkala
Catatan Deployment: Translation Extract adalah utility maintenance yang hanya digunakan saat diperlukan update localization. Tidak memerlukan instalasi khusus di mesin produksi dan cukup dijalankan dari environment development.

Workflow Typical:

  1. Extract new translations dari kode terbaru
  2. Sync translations across semua file bahasa
  3. Separate empty translations untuk memudahkan penerjemahan
  4. Remove unused translations secara periodik untuk cleanup
  5. Deploy hasil update ke server production melalui git workflow
